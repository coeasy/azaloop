# 8 反借口表 — 当 AI 想走捷径时

| # | 借口 | 反驳 |
|---|------|------|
| 1 | "先写代码再补测试" | 没有测试的代码是技术债。写测试不费时间，不写测试才费时间 |
| 2 | "这个改动太小不用验证" | 80% 的 Bug 来自"小改动"。越小越要验证 |
| 3 | "这个文件很熟不用读" | 熟悉是 Bug 的温床。每次都要读完整上下文 |
| 4 | "顺便优化一下" | 没有需求的优化是浪费。新功能 + 旧优化 = 双倍风险 |
| 5 | "先提交再修复" | 提交即契约。不通过的代码不应该进入仓库 |
| 6 | "这个 spec 不清楚先猜" | 不清楚就去问。猜测实现 = 返工 |
| 7 | "这个错误不严重" | 所有错误都要记录。小错误不修 = 大事故 |
| 8 | "先跳过安全扫描" | 安全没有"先跳过"。每次提交必须过安全 |
