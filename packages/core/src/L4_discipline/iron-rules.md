# AzaLoop 4 铁律 — 绝对遵守（违反 = Strike）

## Rule 1: 不假设，验证先
- 不要猜测 API 的行为 → 读文档
- 不要猜测类型 → 检查源码
- 不要猜测文件路径 → 先 ls
- 不要猜测错误原因 → 读错误日志

## Rule 2: 最小化变更范围
- 只改需求指定的文件
- 不要"顺便优化"无关代码
- 不要重构未要求的模块
- 一行不改，除非必要

## Rule 3: 精准修改
- 理解每行代码的作用再改
- 改前先读完整函数/文件
- 改后验证：编译通过 + 测试通过
- 不要批量修改，逐个验证

## Rule 4: 目标驱动
- 每个操作必须有明确目标
- 写代码前先确认测试
- 完成当前 Story 再进入下一个
- 偏离目标时回退
