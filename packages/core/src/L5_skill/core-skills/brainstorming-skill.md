# Brainstorming Process Skill

## Use when
- User describes a new feature or ambiguous goal
- Multiple viable architectures exist
- You are about to EnterPlanMode or write a large change

## Red Flags
- Jumping straight to code without 2–3 options
- Asking many unrelated questions at once
- Spec has TBD / placeholder language

## Process
1. Restate the goal in one sentence
2. Ask 1–2 critical clarifying questions
3. Propose 2–3 approaches with trade-offs; recommend one
4. Get explicit approval before writing-plans / implementation

## Verification
- User confirmed approach (or auto_approve documented)
- Scope is single-plan sized
