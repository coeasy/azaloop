## 四、关键路径分析

**核心依赖链**: P1-1 → P1-2 → P1-3 → P2-1 → P2-5

- P1-1 修复测试是所有后续开发的信心基础
- P1-2 竞品研究集成是 AzaLoop 独特卖点的实证
- P1-3 孤儿模块清理是架构健康的前提
- P2-1 工具分组是上下文优化的核心（直接影响 token 成本）
- P2-5 记忆压缩是长期运营的成本保障

**独立并行链**:
- P2-2 (resume.md) 可与主线并行
- P1-4 (一次性模式) 可立即开始
- P2-3 (CI 安装包) 可在 P1 完成后启动

---

## 五、总结: 未来 6 个月最核心的 3 件事

1. **修复 + 集成（P1-1/P1-2/P1-3）**— 先让已有能力全部贯通，消除测试红灯和孤儿逻辑
2. **上下文优化（P2-1/P2-5）**— MCP 工具分组注册 + 记忆自动压缩，降低 ≥50% token 成本
3. **跨会话可靠性（P2-2/P2-4）**— resume.md 标准化 + runParallel 集成，确保全自动循环在任何客户端都能续航

这三件事完成后，AzaLoop 将从"功能最全但最复杂"转变为"功能最全且最可靠"的 PRD 驱动开发循环引擎。
