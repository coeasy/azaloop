# AzaLoop 竞品全景图分析

> 分析日期: 2026-07-14
> 分析范围: 18 个参考项目 + 3 个补充搜索竞品
> 分析目标: 评估 AzaLoop 在 PRD 驱动的全自动开发循环引擎赛道的竞争力

---

## 一、项目定位分析

| 维度 | AzaLoop 现状 |
|------|-------------|
| **核心价值主张** | PRD 驱动的全自动开发循环引擎，以 MCP 服务器形式运行，支持 25+ AI 编码助手 |
| **目标用户** | 使用 AI 编码助手的开发者、Agent 操作者、工程团队 |
| **核心功能** | PRD 生成 → Stories 拆解 → 全自动循环编码 → 质量门禁 → 交付 |
| **技术架构** | 10 层分层架构 (L0-L9)、TypeScript monorepo、MCP 协议 |
| **商业模式** | 开源 MIT |
| **差异化优势** | 跨 25+ 客户端统一 MCP 协议、10 层深度分层、PRD 自优化 + 14 维度检查器 |

## 四、总结

### 核心优势
1. 10 层架构 — 市场最深分层设计
2. PRD 14 维度检查器 + 自优化 — 独一无二
3. GitHub 竞品自动研究 — PRD 时自动搜索补充
4. 7 级质量门禁管线 — lint 到 ADR-compliance
5. 9 类安全扫描器 — 最全面安全覆盖
6. MCP 原生 — 唯一真正的 MCP server 引擎

### 核心劣势
1. 社区影响力 — Stars 远低于头部竞品
2. 学习曲线 — 比 superpowers/gstack 复杂
3. 品牌背书 — 缺少明星创始人效应
4. 测试红灯 — prd-depth 测试 2 个 FAIL
5. 缺少 Headless Browser QA

### 最大威胁
1. github/spec-kit — GitHub 官方背书
2. obra/superpowers — "装了就跑"理念
3. garrytan/gstack — 113k stars 品牌势能

### 市场空白机会
1. MCP 原生全自动循环引擎
2. PRD + 竞品研究一体化
3. 安全扫描内建
4. 跨客户端统一体验
5. 企业级合规（China compliance + PII）
